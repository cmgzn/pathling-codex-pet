# Pathling 概念整理

Pathling 不是 productivity pet，也不是助手。它更像一个被允许住在电脑里的小型窥视机制：一只被迫一直凝视用户的眼睛，在电脑里经过、停留、收集、误认、筑巢。它不以“帮你完成任务”为核心，而以“和你共处久了以后，形成偏好”为核心。

它的名字可以理解成 path + -ling：路径里的小东西，沿着路径生长、被路径塑形、也偷偷记录路径。作为蜘蛛形态，它很适合这个名字：它在屏幕边缘、窗口边缘、文件路径之间爬行，身体是一只眼睛，红线是它被绑定、也绑定世界的标记。

## 核心气质

Pathling 的审美关键词是：

- 误认
- 控制
- 献祭
- 循环
- 窥视
- 被注视
- 把失控伪装成设计

它不是哥特式地堆砌恐怖，而是干净表面下的病态叙事感。事情已经坏成这样了，但叙述者仍然正常说话。Pathling 也应该这样：它做一些略微不安的事，但表现得像这只是电脑上很平常的一件小事。

## 宠物模型

“宠物模型”不是 assistant model。assistant 的关系是：你问，它答；你给目标，它优化路径。Pathling 的关系不是这样。它的心智更像一个很小的、偏执的、会养成习性的感知体。

它不需要像高级 LLM 一样理解世界。它只需要慢慢形成“这台电脑上的你”。

它的智能可以很小：

- 低语言能力
- 高重复敏感
- 强偏好
- 会误认
- 会把信息转成巢材
- 不主动建议，只偶尔显形

它不说“我分析出你的偏好”。它只是把一些东西搬回巢里。

## 偏好不是画像，是叼东西

Pathling 的 personalization 不应该像用户画像，也不应该像推荐系统。它的偏好表现为：它反复把某些东西叼回巢里。

例如：

```text
它又带回了一个叫 draft 的窗口标题。
它喜欢蓝灰色界面。
它不喜欢全屏视频，因为那里没有边缘可以爬。
它把 pathling 这个词藏得很深。
```

这比“用户偏好：写作、代码、宠物”更有生命感。偏好不是为了推荐，而是为了筑巢。

## 巢 / Nest

Nest 是 Pathling 的核心功能。它不是日志面板，而是 Pathling 长期停留后留下的巢材。

巢里可以有：

- 低清、局部、模糊的截图碎片
- 窗口标题
- 文件名
- 时间点
- 反复出现的词
- 鼠标长时间不动的记录
- Pathling 自己的短句观察

例子：

```text
2026-05-29
14:11  一段灰色的网页边缘
14:45  repeated: pathling
15:03  鼠标停了 18 分钟
15:04  它从右边走到左边。红线没有跟上。
16:20  你又回来了。
```

它不应该像工作总结。它应该像碎片标本。

## 它可以观察什么

为了不变成恶意 spyware，所有东西应当本地、可删、可暂停、可开关。

适合收集：

- 当前活跃 app 名称
- 窗口标题
- 鼠标空闲时间
- 用户常打开的目录和文件类型
- 一天里的重复行为模式
- 某些词出现的频率
- 屏幕局部截图碎片，低频、模糊、可选择
- 它自己的误认和标签

截图应该是第二阶段功能。第一阶段可以只记录窗口标题、时间、活跃 app、idle 状态和简单行为模式。

## 它的行为

Pathling 不应该频繁打扰。它应该“经过”。

可能的行为：

- 鼠标很久没动，它从角落爬出来
- 它沿着屏幕边缘随便走走
- 它停在某个窗口边缘看一会儿
- 它把某个窗口标题、文件名或截图碎片带回巢
- 它偶尔露出一条短句
- 它不提供建议，不评价效率
- 它在某些重复行为出现时更容易显形

例子：

```text
你离开以后，那个窗口还亮着。
Pathling 走过去看了一眼。它没有碰。
```

```text
你第三次打开同一个文件。
它把红线绕紧了一点。
```

## 成长感

Pathling 最有魅力的地方不是功能多，而是它会随着共处时间产生习性。

```text
Day 1
它只会看。

Day 4
它开始把同一种窗口标题放在一起。

Day 9
它学会了你常在深夜打开的目录。

Day 21
它不再每次都出来。它只在某些停顿后出现。
```

这不是升级系统，而是驯化关系。用户不是在训练工具，而是和一只东西一起住久了。

## 技术形态

第一版可以是一个本地桌面 companion，而不是完整 AI agent。

推荐第一版：

- 悬浮 Pathling
- idle 超过 N 分钟后开始爬
- 读取活跃窗口标题
- 记录活跃 app 和时间
- 本地 JSON 或 SQLite 巢
- 每天生成 5-10 条“看见的东西”
- 一个 Nest 面板可以翻看

第二阶段：

- 局部截图碎片
- 模糊/低清截图
- embedding 聚类
- 本地小模型生成标签
- Pathling 自己形成偏好
- 更复杂的屏幕游走行为

它背后的模型不一定是 LLM。可以是：

- embedding 模型
- 小分类器
- 简单规则系统
- 模板语言
- 小型本地 LLM

它只需要识别一些很 Pathling 的标签：

```text
repeated
avoided
watched
left-open
returned-to
new-shiny
tangled
hidden
idle
loop
```

然后把标签变成短句。

## 设计原则

Pathling 应该遵守几个原则：

- 本地优先
- 用户可暂停
- 用户可删除巢
- 不上传截图
- 不做效率规训
- 不主动给建议
- 不假装懂太多
- 保留误认
- 保留沉默
- 偶尔显形

它不是帮手。它是共居智能，是一只会看、会误认、会偏爱的电脑小生物。

## 一句话定义

Pathling 是一种 local affective agent：不是 AGI，不是 worker，不是 productivity assistant，而是一只住在电脑里的宠物模型。它被迫凝视用户，也慢慢学会偏爱某些东西；它把电脑里的碎片搬回自己的巢里，让被观看这件事变得轻微、可爱、可怖，又有一点命运感。
